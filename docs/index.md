# Logs

## start with templates via CLI

```bash
# Initialize a serverless application with a SAM template
sam init \
    --runtime python3.8 \
    --app-template hello-world \
    --package-type Zip \
    --name sandbox-sam-python
cd sandbox-sam-python

# Initialize a Git repository
git init
git add .
git commit -m "sam init"
git branch -m main

# Create GitHub repository
gh repo create sandbox-sam-python -y --public

git push -u origin main
```

## SAM CLI

- [AWS SAM CLI 再入門 2020.07](https://qiita.com/hayao_k/items/7827c3778a23c514e196)
- [How do I destroy a aws SAM Local lambda?](https://stackoverflow.com/a/50888806)

```bash
sam validate

sam build
sam local invoke
sam local generate-event dynamodb update > ./events/dynamodb_event.json

# Starts a local endpoint to invoke your local Lambda functions
sam local start-lambda
aws lambda invoke --function-name HelloWorldFunction  --endpoint http://127.0.0.1:3001/

# Sets up a local endpoint to test your API
sam local start-api
curl http://127.0.0.1:3000/hello

# creates a Cloudformation Stack and deploys your resources
aws s3 help
aws s3 mb s3://sandbox-sam-python
sam deploy --guided
sam deploy --stack-name sandbox-sam-python --s3-bucket sandbox-sam-python --region ap-northeast-1 --capabilities CAPABILITY_IAM --confirm-changeset
curl https://xxxxxxxxxx.execute-api.ap-northeast-1.amazonaws.com/Prod/hello/

# publish a packaged AWS SAM template to the AWS Serverless Application Repository
sam publish --help

# fetch logs generated by your Lambda function
sam logs -n HelloWorldFunction --stack-name sandbox-sam-python

# destroy the stack
#   `sam destroy` command is work-in-progress for now
#     - https://github.com/aws/aws-sam-cli/issues/2191
#   just run aws CLI commands below
#     - https://stackoverflow.com/questions/48324894/how-do-i-destroy-a-aws-sam-local-lambda
aws cloudformation delete-stack --stack-name sandbox-sam-python
```
